# 今日一号令

> 一个只记录落实的个人工作记录本。
> 不帮你安排更多事情，只帮你把眼前的一件事做清楚。

“今日一号令”适合用来处理那些你已经知道应该做、但容易被焦虑、分心或任务过载拖住的事情。

它不要求你成为更自律的人。它只尝试帮你把问题从：

> 我做不到，所以我很差。

改写成：

> 现在的问题是什么？
> 已经完成了什么？
> 还缺什么？
> 下一步怎么接上？

---

## 它从哪里来？

这个工具受到一些工作方法的启发：

* 先看事实，再下判断
* 复杂问题先列纲目
* 抓住当前的主要矛盾
* 决定之后要落实，落实之后要检查
* 每完成一次重要工作，就总结能被下一次复用的方法

它不是为了复刻任何人的工作方式，而是尝试把“调查、判断、部署、落实、复盘”变成一个可以用于个人生活和工作的轻量记录系统。

---

# 快速开始

每天只需要完成三个动作：

```text
下达今日一号令 → 留下事实 → 晚上查令
```

## 1. 下达今日一号令

写下两件事：

* 今天最重要的一件事是什么？
* 做到什么程度才算完成？

例如：

```text
中心任务：完成项目方案的第一版
完成标准：整理成 PDF，并发送给合作方确认
```

也可以是：

```text
中心任务：复习统计学第五章
完成标准：完成课后习题 1–10，并整理错误题目
```

重点不是写“推进项目”“努力学习”“处理工作”。

而是写一个能够被清楚判断是否完成的结果。

---

## 2. 留下事实

开始或执行过程中，记录：

* 实际开始时间
* 实际完成物
* 遇到的问题和当前解决方法

例如：

```text
实际开始时间：14:30
实际完成物：完成方案初稿，共 12 页
遇到的问题和当前解决方法：案例部分资料不足；先使用现有资料完成框架，明天补充两个案例
```

这里不需要写“今天状态很差”“我是不是不适合做这件事”。

尽量只记录事实、结果、问题和当前方案。

---

## 3. 晚上查令

收工时回答三个问题：

```text
今日令完成了吗？
没完成的事实原因是什么？
明日从哪里接上？
```

例如：

```text
今日令完成了吗：完成 80%
没完成的事实原因：案例部分资料还没有整理完
明日处理：先补充两个案例，再统一格式并发送
```

查令不是自我批评。

它的作用是让任务不断线。

# 其他页面什么时候使用？

## 每周复盘

每周一次，按五个方面回顾：

* 事业
* 经济
* 身体
* 人际
* 当前斗争

“当前斗争”指的是：

> 此刻真正阻碍你推进的主要矛盾是什么？

例如：

```text
我同时想推进很多事情，导致每一件都只开始了一点。
```

或者：

```text
我需要完成一个重要任务，但一直在用低优先级的小事逃避它。
```

每周复盘不要求你给自己打分。

它只帮助你发现：哪一部分失衡了，下周应该把资源放在哪里。

---

## 一号工程

当你进入一个重要阶段时使用。

例如：

* 完成一个毕业设计或研究项目
* 准备一场考试
* 找工作或准备申请材料
* 做一个产品或作品集
* 筹备一次活动
* 建立一个长期习惯
* 完成一轮内容创作计划

开始时，写清楚：

```text
为什么现在必须做它？
完成标准是什么？
预计周期是什么？
每周节点是什么？
这段时间明确不做什么？
```

结束后，复盘：

```text
实际完成了什么？
哪些方法有效？
哪些方法失败？
以后可以推广成什么工作方法？
下一个一号工程是什么？
```

一号工程的作用是：在一段时间里，明确什么是最值得集中资源完成的事。

---

## 新赛道调研

准备进入一个陌生领域时使用。

例如：

* 转行
* 学习一项新技能
* 准备进入一个新行业
* 做一个从未做过的产品
* 开始运营一个新账号
* 申请一个陌生岗位或项目

它帮助你先弄清：

```text
这个领域真正奖励什么？
成功案例为什么成功？
失败案例为什么失败？
我的优势和短板是什么？
我不应该硬拼什么？
我的第一场小胜利应该是什么？
```

先调研，再行动。

不要只靠想象进入一个新领域。

---

## 动态竞争

适合比赛、投稿、竞标、申请、产品发布、求职、创业等会持续变化的项目。

每个阶段记录：

```text
阶段成果
外部反馈
形势变化
当前主要矛盾
调整部署
下一检查点
```

初始计划不是最终计划。

当反馈、资源、竞争环境或目标发生变化时，可以不断调整路径，但始终盯住最终目标。

# 数据如何保存？

本工具不需要注册，也不会把你的记录上传到服务器。

你的数据默认只保存在：

> 当前设备 + 当前浏览器

例如，你在 Chrome 中填写的记录，不会自动出现在 Edge、Safari 或另一台电脑上。

---

## 日常使用时，数据会自动保存吗？

会。

每次点击“保存每日记录”“保存每周复盘”等按钮后，记录都会保存到当前浏览器。

你下次用同一台电脑、同一个浏览器、打开同一个网址时，历史记录仍会存在。

---

## 如何备份数据？

建议每周备份一次，或者在完成一个重要的一号工程后备份一次。

步骤：

```text
1. 打开网页顶部的“导出 JSON 备份”
2. 浏览器会下载一个 .json 文件
3. 把这个文件放到你自己知道的位置
   例如：Google Drive、iCloud、OneDrive、电脑文件夹或移动硬盘
```

JSON 文件是完整备份。

它可以用来恢复全部记录。

---

## 如何恢复数据？

当你换了电脑、换了浏览器，或担心数据丢失时：

```text
1. 打开“今日一号令”
2. 点击顶部的“导入 JSON 备份”
3. 选择之前导出的 .json 文件
4. 导入后，历史记录会恢复
```

---

## 如何把记录导出成可阅读文件？

点击：

```text
导出全部 Markdown
```

浏览器会下载一个 `.md` 文件。

这个文件适合：

* 长期归档
* 复制到 Notion、Obsidian、语雀等工具
* 保存某个阶段的复盘
* 阅读自己的历史记录
* 整理成年终总结或项目总结

---

## 使用时请注意

```text
不要使用无痕模式
不要随意清理浏览器网站数据
不要只依赖本地保存
定期导出 JSON 备份
换设备前先导出，再导入
```

# 把它改成你的版本

《今日一号令》不是一套需要严格服从的标准答案。

它最初只是为了解决一个很具体的问题：当任务、焦虑和自我评价混在一起时，怎样把注意力重新放回事实、资源、方案和下一步行动。

你完全可以把它改造成更适合自己的记录工具。

例如：

* 觉得“每日一号令”太少，可以改成“今日三件要事”
* 不需要“经济”或“人际”栏目，可以替换成家庭、创作、学习、关系、信仰或任何你真正关心的部分
* 觉得“当前斗争”太严肃，可以改叫“当前卡点”“本周核心问题”或“最需要处理的事”
* 可以加入习惯、睡眠、阅读、运动、情绪、创作灵感或项目进度
* 可以删掉自己永远不会填写的模块
* 可以把页面改成更适合手机、更像日记、更像项目看板，或更像一张纸质工作单
* 可以替换成任何你从一本书、一段经历、一种工作方法里提炼出的个人系统

这个项目最想表达的不是：

> 每个人都应该使用同一种记录方式。

而是：

> 每个人都可以把真正对自己有用的思想方法，改造成一个每天愿意打开、能够帮助自己行动的工具。

欢迎 Fork、复制、修改和重做。

希望你最后得到的不是“另一个效率工具”，而是一份越来越了解自己的使用说明书。

---

# 它不是什么

它不是：

* 待办清单
* 打卡工具
* 情绪日记
* AI 教练
* 自动规划系统
* 复杂项目管理软件

它只是一个让你把工作从“对自己的审判”，变成“一个可以观察、拆解和继续推进的问题”的工具。

> 今天最重要的一件事，是什么？
