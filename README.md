# 今日一号令

一个基于《毛泽东选集》方法论的个人日程管理工具。

它把实事求是、调查研究、抓主要矛盾和“实践—认识—再实践”等方法，转化为可执行的个人计划、行动记录与复盘流程：每日确定中心任务，以事实验收结果；每周分析主要矛盾；在“一号工程”结束后沉淀可复用的方法。

> 本项目是非官方个人工具，不代表任何机构立场。项目不提供《毛泽东选集》原文，仅做方法论层面的产品化表达。

## 功能

- 每日：确定中心任务、完成标准、证据记录、晚间查令、学习记录
- 每周：事业、经济、身体、人际、当前斗争五纲复盘，识别主要矛盾
- 一号工程：启动记录与结束复盘
- 新赛道调研与取胜战略
- 动态竞争记录
- 本地自动保存、JSON 备份与 Markdown 导出

## 方法映射

| 方法 | 在工具中的落地 |
| --- | --- |
| 实事求是 | 用“实际产出”和证据验收任务，不以感觉代替事实 |
| 抓主要矛盾 | 每日只确定一个中心任务，每周重新判断主要矛盾 |
| 调查研究 | 在新赛道调研中记录规则、样本、优势与短板 |
| 实践与认识循环 | 执行、复盘、修正判断，再形成下一步行动 |
| 集中优势力量 | 用“一号工程”管理阶段性最重要的目标 |

## 部署到 GitHub Pages

1. 新建一个 GitHub 仓库，例如 `today-first-order`。
2. 上传本项目中的全部文件，尤其是 `index.html`。
3. 进入仓库 `Settings` → `Pages`。
4. 在 `Build and deployment` 中选择：
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/(root)`
5. 点击 `Save`，等待部署完成。

网站地址通常为：

`https://<你的GitHub用户名>.github.io/today-first-order/`

## 数据说明

本项目使用浏览器 localStorage 保存记录：

- 同一浏览器、同一网址下可持续保存。
- 不会自动同步到其他设备。
- 请定期使用“导出 JSON 备份”。
- 从本地 HTML 迁移到 GitHub Pages 前，先导出 JSON；上线后打开 Pages 网站，再导入 JSON。

## 本地运行

直接双击打开 `index.html` 即可。
